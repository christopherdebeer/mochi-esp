# Toolchain setup (repeatable)

One command brings a fresh checkout — laptop, CI, or a Claude Code
remote/web session — to a buildable state:

```sh
firmware/scripts/setup-esp-idf.sh
```

This clones **ESP-IDF v5.3** (the verified version) and installs the
`xtensa-esp32s3-elf` toolchain into `~/.espressif` (~2 GB, first run
only). It is safe to re-run — an existing ESP-IDF checkout is reused and
`install.sh` is idempotent.

Then, in each shell that builds:

```sh
. ~/esp/esp-idf/export.sh        # activate ESP-IDF
cd firmware
idf.py set-target esp32s3        # one-time per checkout
idf.py reconfigure               # one-time per clean clone — fetches managed_components/
idf.py build
```

Override the install location or IDF version via env vars:

```sh
IDF_BASE=/opt/esp-idf IDF_BRANCH=release/v5.4 firmware/scripts/setup-esp-idf.sh
```

## Notes for remote / web sessions

- The toolchain lives outside the repo (`~/esp`, `~/.espressif`) and is
  **not** committed, so it must be re-installed when a fresh container
  starts. Run the script once per new environment before building.
- `managed_components/` is `.vtignore`d and regenerated by
  `idf.py reconfigure` from `dependencies.lock` + `idf_component.yml`.
- For flashing/monitoring on real hardware and troubleshooting, see
  [`firmware/README.md`](README.md).
