# Firmware — ESP-IDF project for ESP32-S3-Touch-ePaper-1.54 V2

The firmware is an ESP-IDF project. Check it out locally (or `vt clone`
the val) and build with `idf.py`. The source lives next to the design
docs so the two stay in sync.

## Toolchain

ESP-IDF **v5.3 LTS** verified. v5.2 / v5.4 likely work but aren't
exercised. The `i2c_master` driver and clock-stretch tolerance behaviour
in particular are v5-era — see `../design/01-bring-up-plan.md` M8 for
why polling-mode SHTC3 commands are mandatory on v5.

### macOS / Linux

```sh
mkdir -p ~/esp && cd ~/esp
git clone -b release/v5.3 --recursive https://github.com/espressif/esp-idf.git
cd esp-idf
./install.sh esp32s3        # macOS / bash / zsh
./install.fish esp32s3      # if you use fish

# every shell that builds:
. ~/esp/esp-idf/export.sh
```

`install.sh` downloads the xtensa-esp32s3-elf toolchain into
`~/.espressif/`. ~2 GB total.

### Windows

ESP-IDF Tools Installer from <https://dl.espressif.com/dl/esp-idf/>
bundles git, python, and the toolchain. After install, use the "ESP-IDF
PowerShell" or "ESP-IDF Command Prompt" shortcut for a pre-configured
environment.

## Build, flash, monitor

From `firmware/` after `. export.sh`:

```sh
idf.py set-target esp32s3        # one-time per checkout
idf.py reconfigure               # one-time per clean clone — fetches managed_components/
idf.py build
idf.py -p /dev/cu.usbmodem* flash monitor
```

- macOS: `/dev/cu.usbmodem*` — `ls /dev/cu.*` with the cable plugged in
- Linux: typically `/dev/ttyACM0`
- Windows: `COMn` — Device Manager

Exit `idf.py monitor` with `Ctrl-]`.

### Recovering a bricked device

If a flashed image crashes early enough to prevent the next flash (we hit
this once during M8.5 stack-overflow debugging):

1. Hold **BOOT** while plugging in USB-C → ESP32-S3 enters ROM download
   mode regardless of firmware state.
2. `idf.py -p /dev/cu.usbmodem* flash` to write a known-good image.
3. The on-device factory reset (PWR + BOOT held 10 s) wipes NVS but
   needs working firmware to run, so it doesn't help with brick recovery.

`sdkconfig` is generated by `idf.py menuconfig` and **intentionally not
committed**. `sdkconfig.defaults` is the source of truth; if you need a
config tweak, edit defaults and run `idf.py reconfigure`. Delete
`sdkconfig` to force regen if it gets out of sync.

`managed_components/` is regenerated from `dependencies.lock` +
`idf_component.yml` on a clean clone. `idf.py reconfigure` does this in
one shot. A clone is build-ready after one `reconfigure`.

## Current state — M8.5 ("Feels like Mochi")

The minimum spine is done and the device is interactive: pet-on-scene
compositing, 5-zone touch UI with care icons, status bar (time + name +
battery), persistent sprite cache with server-driven ETag invalidation,
deep-sleep gesture, factory reset, battery sense.

Boot is currently:
1. NVS WiFi creds present? → join + WAITING-FOR-PAIRING (M3) or pair-check (M5)
2. NVS pet_id present? → render WARM-BOOT scene from cache
3. Cache miss / ETag stale? → HTTPS fetch + write back

See `../design/02-boot-sequence.md` for the full state machine and
`../design/01-bring-up-plan.md` for the milestone history.

`main.cpp` is the orchestration layer; if you're adding a new boot
branch, that's the first place to look. Per-peripheral logic goes in its
own `.cpp/.h` pair under `main/`.

## Hardware reference

Waveshare ESP32-S3-Touch-ePaper-1.54 V2.

- Wiki + schematic: <https://www.waveshare.com/wiki/ESP32-S3-Touch-ePaper-1.54>
- Pin map: `main/board_pins.h` (authoritative; recovered from
  `user_config.h` of `waveshareteam/ESP32-S3-ePaper-1.54` since the
  wiki PDF was unavailable at bring-up)

Notable pins:

- LED: GP3 (also a strapping pin — fine for runtime output)
- ADC battery sense: GP4 (ADC1 ch3, 1:2 divider on VBAT_PWR)
- I²C: SDA/GP47, SCL/GP48 (touch + RTC + SHTC3 share the bus)
- Touch: INT/GP21, RST/GP7
- **Audio_PWR: GP42** — *named for the codec but actually feeds all I²C
  peripherals.* Must be enabled before any I²C bring-up. (Project memory
  `project-eink-power-rails`.)
- 12-pin header: RXD/TXD, SDA/GP20, SCL/GP19, GND/GP3, 3V3/GP2, VSYS/GP1
- BOOT: GP0 (also "PWR" in our 5-zone UI lexicon for the "press 3 s to
  sleep" / "PWR + BOOT 10 s" gestures; not to be confused with a separate
  power switch — the board has only USB-C and the BOOT button)

## Troubleshooting

**Build fails on a clean clone with "component not found"** — run
`idf.py reconfigure` to fetch `managed_components/`.

**Device unresponsive without USB; ADC reads ~4150 mV** — likely the
"with LiPo" SKU shipped with the cell connector empty. Open the case and
verify a battery is actually plugged in. The reading is a leakage path
through the ETA6098 charge IC, not a real cell. (Project memory
`project-eink-missing-battery`.)

**TLS / HTTPS code crashes app_main with stack overflow** — bump
`CONFIG_ESP_MAIN_TASK_STACK_SIZE` to 8192. Default 3.5 KB collides with
mbedTLS bignum frames. Already set in `sdkconfig.defaults`; re-check if
you customise. (Project memory `project-eink-main-stack`.)

**`ESP_ERR_INVALID_STATE` from an I²C read** — slow sensors with
clock-stretching exceed ESP-IDF v5's default tolerance. Use polling
variants (e.g. SHTC3 cmd `0x7866` not `0x7CA2`). (Project memory
`project-eink-i2c-stretching`.)

**`esp_http_client_get_header()` returns nullptr after `perform()`** —
v5.3 quirk on HEAD requests over chunked / HTTP2-fronted backends.
Capture inside the `HTTP_EVENT_ON_HEADER` callback instead, with
`strcasecmp` on the header name.

**Serial doesn't appear after flash** — `sdkconfig.defaults` enables
`CONFIG_ESP_CONSOLE_USB_SERIAL_JTAG=y`. If you've set it to UART, the
console goes to GPIO43/44 instead. Re-flip and rebuild.

**`idf.py flash` says "wrong chip"** — `idf.py set-target esp32s3` and
rebuild.

**PSRAM not detected** — N8R8 is octal PSRAM; defaults set
`CONFIG_SPIRAM_MODE_OCT=y`. On a different board with quad PSRAM, flip
to `CONFIG_SPIRAM_MODE_QUAD=y`.

**WiFi mode swap deadlocks during AP→STA hand-off** — never swap modes
inline; persist creds, `esp_restart()`, take the STA branch on the next
boot. (Project memory `project-eink-wifi-handover`.)
